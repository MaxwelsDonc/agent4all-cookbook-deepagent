# Agent4all

> 让 DeepAgent 不再 "deep"。

Agent4all 是一个面向新手的 DeepAgent 开源教程项目。

这个项目想做一件很简单的事：把看起来很深、很复杂、很容易劝退人的 DeepAgent 学习路径，拆成普通人也能看懂、跑起来、改得动的内容。

很多人第一次接触 Agent 时，会同时遇到一堆概念：大模型、提示词、工具调用、记忆、工作流、API、框架、部署、成本控制。Agent4all 希望把这些东西按学习顺序重新整理，让小白可以从 0 开始，用尽量低的成本理解 Agent 到底是什么，以及它能帮自己做什么。

## 这个项目会关注什么

- 用新手能听懂的方式解释 DeepAgent 和 AI Agent。
- 尽量提供 0 成本或低成本的学习路径。
- 优先使用开源工具、免费额度、本地环境和可替代方案。
- 把复杂概念拆成一步一步可理解、可复现的实践。
- 记录真实踩坑过程，而不是只展示跑通后的完美结果。

## 适合谁

- 想学习 DeepAgent，但不知道从哪里开始的人。
- 没有太多编程基础，但愿意动手尝试的人。
- 想用 AI Agent 提升学习、办公或自动化效率的人。
- 希望低成本探索大模型应用的人。
- 想从教程开始，慢慢做出自己 Agent 项目的人。

## 为什么叫 Agent4all

Agent 不应该只是少数工程师和研究者才能理解的东西。

Agent4all 的意思是：Agent for all。希望这个项目能让更多普通人理解 Agent、使用 Agent，并最终创造自己的 Agent。

## 本地预览和调试网页

这个项目使用 VitePress 搭建文档站。

第一次运行前，先安装依赖：

```bash
npm install
```

启动本地开发服务：

```bash
npm run docs:dev
```

启动后，终端会显示一个本地地址，通常是：

```text
http://localhost:5173/agent4all-cookbook-deepagent/
```

在浏览器打开这个地址，就可以预览网页。

修改 `docs/` 里的 Markdown 文件后，页面通常会自动热更新。如果浏览器没有变化，可以手动刷新页面。

提交前建议运行一次构建检查：

```bash
npm run docs:build
```

如果构建通过，说明当前文档站可以正常生成静态网页。
