# Planning：先想清楚再行动

## 这一节你会学到什么

你会理解 Planning 是什么，以及为什么复杂任务不能一上来就让 Agent 直接回答。

你还会认识 Deep Agents 里一个很重要的内置能力：`write_todos`。

## 一句话讲清楚

Planning 就是让 Agent 先把大任务拆成小步骤，再一步一步完成。

用“电脑里的人”来理解：

> 你不是让他一听到任务就立刻开写，而是先让他拿出纸，列一张待办清单。

## 为什么需要它

简单任务可以直接回答。

比如：

```text
用一句话解释什么是 Agent。
```

这种任务没有太多步骤，直接回答就行。

但复杂任务不一样。

比如：

```text
帮我制定一个 7 天 DeepAgent 学习计划，
每天安排学习内容、练习任务、复盘问题，
最后再生成一份检查清单。
```

这个任务至少包含：

- 判断读者当前水平。
- 分配 7 天学习内容。
- 给每天安排练习。
- 设计复盘问题。
- 生成检查清单。
- 确认输出是否完整。

如果 Agent 不先拆任务，它很容易漏掉某一步。

## 用一个简单例子理解

假设你请一个同学帮你搬家。

如果你只说：

```text
帮我搬家。
```

他可能不知道先做什么。

更靠谱的做法是先列步骤：

```text
1. 列出要搬的东西。
2. 准备纸箱。
3. 先打包书和衣服。
4. 标记易碎物品。
5. 联系车。
6. 搬到新家后检查有没有遗漏。
```

Agent 做复杂任务也是一样。

Planning 不是为了让输出看起来更正式，而是为了减少遗漏。

## 回到 Agent

Deep Agents 默认适合处理更长的任务，其中一个原因就是它有任务规划能力。

你会在官方资料里看到 `write_todos`。

它可以先理解成：

> Agent 给自己写待办清单的工具。

这个工具不是给用户看的普通 Markdown 列表，而是 Agent 执行任务时用来跟踪进度的结构化清单。

一个待办通常会有几种状态：

```text
pending：还没开始
in_progress：正在做
completed：做完了
```

这能帮助 Agent 知道：

- 当前做到哪一步。
- 后面还剩什么。
- 有没有任务被遗漏。

## 代码里长什么样

先看一个最小示例。

在项目根目录创建一个新文件：

```text
complex_agent.py
```

写入下面代码：

```python
from dotenv import load_dotenv
from deepagents import create_deep_agent

load_dotenv()

agent = create_deep_agent(
    model="openai:gpt-4o-mini",
    system_prompt=(
        "你是一个耐心的 DeepAgent 学习教练。"
        "遇到复杂学习任务时，你要先拆成清晰步骤，再逐步完成。"
        "如果任务超过 3 步，请先规划再回答。"
    ),
)

result = agent.invoke({
    "messages": [
        {
            "role": "user",
            "content": (
                "请帮我制定一个 7 天 DeepAgent 学习计划。"
                "每天要包含学习主题、动手练习和复盘问题。"
            ),
        }
    ]
})

for message in result["messages"]:
    print("\n---", message.type, "---")
    print(getattr(message, "content", ""))
```

这段代码里没有自定义工具。

但 Deep Agents 本身会给复杂任务准备一些内置能力。不同版本和不同模型的输出细节可能不完全一样，你现在先关注“它有没有先组织步骤”。

## 跟着做

确认你已经完成第二章的准备：

- 虚拟环境已经激活。
- `.env` 里有 API Key。
- 已经安装 `deepagents`、`python-dotenv` 和模型 provider 包。

然后运行：

```bash
python complex_agent.py
```

## 你应该看到什么结果

你应该看到一段比较长的输出。

里面可能包含：

- 用户消息。
- Agent 的规划过程。
- 最终的 7 天学习计划。
- 某些环境下可能出现和待办、工具调用有关的信息。

不用要求每次输出和教程完全一样。

你只需要检查三件事：

```text
1. 它有没有把任务拆开？
2. 7 天计划有没有覆盖学习主题、练习和复盘？
3. 输出有没有明显漏掉用户要求？
```

如果这三件事都满足，这一节就跑通了。

## 常见错误

### 错误 1：以为 Planning 就是多写几个标题

不是。

真正的 Planning 是为了帮助 Agent 执行任务，而不是给回答加装饰。

如果计划和最终结果没有关系，那只是表面上好看。

### 错误 2：任务太简单也强行规划

如果只是问：

```text
什么是 token？
```

通常不需要复杂规划。

Planning 适合多步骤、长时间、容易遗漏的任务。

### 错误 3：看不到 `write_todos` 就以为失败

不一定。

不同模型、不同任务、不同版本的 Deep Agents，输出里展示的细节可能不一样。

入门阶段先看结果是否真的被拆解和完成。后面调试时，再进一步看工具调用和执行轨迹。

### 错误 4：模型直接回答，没有明显拆步骤

可以把用户任务写得更明确：

```text
这是一个复杂任务。请先列出执行步骤，再逐步完成。
```

也可以在 `system_prompt` 里强调：

```text
复杂任务必须先规划。
```

## 小结

这一节你只需要记住：

```text
Planning：先拆任务，再执行任务。
write_todos：Deep Agents 里帮助 Agent 维护待办清单的能力。
```

复杂 Agent 的第一步，不是让它更会说，而是让它先想清楚怎么做。
