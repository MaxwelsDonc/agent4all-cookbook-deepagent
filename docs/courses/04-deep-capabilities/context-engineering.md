# Context engineering：让 Agent 看见该看的信息

## 这一节你会学到什么

你会理解 Context engineering 是什么。

你还会知道为什么复杂任务不能把所有内容都塞进 prompt 或 messages。

## 一句话讲清楚

Context engineering 就是管理 Agent 当前能看到什么。

用“电脑里的人”来理解：

> 他的桌面空间有限。真正需要马上看的资料放桌面上，暂时不用的资料放文件夹里。

## 为什么需要它

第一章讲过 context：

```text
Context 是 Agent 当前能看到、能参考的信息。
```

很多新手会误以为：

> 给 Agent 的信息越多越好。

其实不是。

上下文太乱，会带来几个问题：

- 关键要求被淹没。
- 模型读起来更慢。
- token 成本更高。
- 很长的工具结果会挤占真正重要的信息。
- Agent 更容易忘记当前目标。

所以复杂 Agent 不能只问：

```text
我要给它多少信息？
```

还要问：

```text
这一步它真正需要看见什么？
哪些内容可以先保存到文件？
哪些内容可以总结后再放回来？
```

这就是 Context engineering。

## 用一个简单例子理解

假设你要写一篇读书报告。

你的桌面上可以放：

- 这篇报告的题目。
- 文章大纲。
- 当前正在引用的几段笔记。
- 检查清单。

但不应该把所有书、所有网页、所有历史聊天记录都摊在桌面上。

你可以把大段资料放进文件夹，需要时再拿出来。

Agent 也是这样。

## 回到 Agent

Deep Agents 的上下文管理，核心思想可以先理解成：

```text
不要把所有东西都放进模型当前输入。
```

它会围绕几类信息做管理：

| 信息 | 应该怎么处理 |
| --- | --- |
| 当前用户任务 | 必须放进 context |
| system prompt | 必须放进 context |
| 最近关键对话 | 通常需要放进 context |
| 大段工具结果 | 更适合保存到文件或被总结 |
| 长期偏好 | 更适合放进 memory |
| 子任务细节 | 可以交给 subagent 隔离处理 |

这也是为什么第四章会把文件系统、memory 和 subagent 放在一起讲。

它们都在解决同一个大问题：

> Agent 做长任务时，怎么不被越来越多的信息拖乱？

## 代码里长什么样

继续使用上一节的 `complex_agent.py`。

这一次我们让 Agent 先保存一份计划，再根据这份计划生成复盘问题。

你可以把用户消息改成：

```python
result = agent.invoke({
    "messages": [
        {
            "role": "user",
            "content": (
                "请做三件事："
                "第一，制定一个 7 天 DeepAgent 学习计划；"
                "第二，把计划保存成 deepagent-7-day-plan.md；"
                "第三，只在最终回答里给我 5 个复盘问题，"
                "不要把完整计划重复粘贴一遍。"
            ),
        }
    ]
})
```

这段请求的重点是：

```text
完整计划保存到文件。
最终回答只展示复盘问题。
```

这就是一种很简单的 context 管理练习。

不是所有内容都必须出现在最终回答里。

## 跟着做

修改 `complex_agent.py` 里的用户消息后，再运行：

```bash
python complex_agent.py
```

然后检查两件事：

1. `notes/deepagent-7-day-plan.md` 里是否有完整计划。
2. 终端最终回答是否主要是复盘问题，而不是把完整计划再贴一遍。

## 你应该看到什么结果

理想情况下，你会看到：

```text
文件：保存完整学习计划
终端：只输出 5 个复盘问题
```

这代表 Agent 开始有一点“整理桌面”的意识：

- 大内容放文件。
- 当前需要看的重点放回答。

## 常见错误

### 错误 1：以为 context 就是聊天记录

聊天记录是 context 的一部分，但不是全部。

工具结果、文件内容、记忆、系统提示词，都可能进入 context。

### 错误 2：把所有文件内容都塞进用户消息

如果文件很短，可以直接贴。

但如果文件很长，更好的方式是让 Agent 读取需要的部分，或者先总结再处理。

### 错误 3：最终回答过长

可以明确要求：

```text
完整内容保存到文件，最终回答只给摘要和下一步。
```

这能帮助 Agent 控制输出。

### 错误 4：以为 context engineering 是高级优化，入门不用管

只要任务变长，你就已经在面对 context 问题。

入门阶段不需要掌握复杂算法，但要养成一个习惯：

> 当前这一步，Agent 到底需要看见什么？

## 小结

这一节你只需要记住：

```text
Context engineering：让 Agent 看见该看的信息，而不是看见所有信息。
```

文件系统可以帮你把大段内容放到“桌面外面”。

下一节，我们会讲哪些信息不只是暂时保存，而是应该长期记住。
