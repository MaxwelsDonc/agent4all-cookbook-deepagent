# 看懂第一次输出

## 这一节你会学到什么

你会学习如何看懂 `agent.invoke()` 返回的结果。

第一次输出可能很长，看起来有点吓人。

但你不需要每个字段都看懂。

## 一句话讲清楚

第一次看输出，只先找三件事：用户说了什么、Agent 回了什么、有没有调用工具。

## 为什么输出这么长

Agent 不是只返回一句话。

它可能会包含：

- 用户消息。
- Agent 消息。
- 工具调用记录。
- 工具返回结果。
- 最终回答。

这些内容都属于一次执行过程的一部分。

## 回到第一章

第一章说过：

```text
Messages：聊天记录
Tools：Agent 能用的工具
Context：Agent 当前能看到的信息
```

现在你看到的输出，就是这些概念第一次落到真实结果里。

## 先看最重要的部分

如果你直接 `print(result)`，可能会看到一个比较大的结构。

先找里面的：

```text
messages
```

`messages` 里通常会有多条消息。

你可以先粗略理解成：

| 消息类型 | 可以怎么理解 |
| --- | --- |
| user | 你说的话 |
| assistant | Agent 的回答或下一步动作 |
| tool | 工具返回的结果 |

## 更清楚地打印最后一条消息

为了更容易看结果，可以把上一节代码最后一行：

```python
print(result)
```

改成：

```python
print(result["messages"][-1].content)
```

然后再运行：

```bash
python first_agent.py
```

## 你应该看到什么结果

这次输出会更短。

你应该能看到 Agent 用自然语言回答你，类似：

```text
当然可以。下面是一个 3 天入门计划...
```

具体内容可能不一样。

只要它确实在回答你的问题，就说明你已经看到了最终输出。

## 怎么判断有没有调用工具

因为我们给了 Agent 一个工具：

```python
tools=[make_study_plan]
```

Agent 可能会决定调用它，也可能直接回答。

如果完整输出里出现类似工具调用、工具消息、`make_study_plan` 的内容，说明它调用了工具。

如果没有，也不一定代表失败。

Agent 会根据模型判断决定是否调用工具。

第三章会专门讲工具调用，到时候我们会更稳定地观察工具是怎么被调用的。

## 常见错误

### 错误 1：以为输出不一样就是错了

模型输出不是固定答案。

同一个问题，不同时间可能回答不同文字。

只要没有报错，并且回答和你的问题相关，就算成功。

### 错误 2：一开始就想看懂所有字段

不用。

第一次只看：

- 用户消息。
- 最终回答。
- 有没有明显的工具调用。

其他字段后面会慢慢讲。

### 错误 3：工具没有调用就以为失败

不一定。

Agent 可能判断不需要工具，也可能模型没有选择调用工具。

这正是第三章要解决的问题：让你理解 Agent 为什么、什么时候、怎么调用工具。
