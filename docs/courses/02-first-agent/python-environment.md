# 准备 Python 环境

## 这一节你会学到什么

你会先确认自己的电脑能不能运行 Python，并创建一个虚拟环境。

如果你完全没用过终端，不用慌。

这一节不会让你写复杂代码，只会让你做几件基础准备。

## 一句话讲清楚

Python 是我们写 Agent 示例代码的语言，虚拟环境是给这个项目单独准备的“小房间”。

## 为什么需要它

后面我们会安装 `deepagents` 这样的 Python 包。

如果直接把所有包都装到电脑全局环境里，时间久了容易乱：

- 这个项目需要一个版本。
- 另一个项目需要另一个版本。
- 你也不知道某个包到底是为哪个项目装的。

虚拟环境就是给当前项目准备一个独立空间。

用“电脑里的人”来理解：

> 你不是把工具随便丢在整个房子里，而是给这个学习助手准备一个专属工作间。

## 终端是什么

终端可以理解成你和电脑直接对话的窗口。

你在终端里输入命令，电脑执行命令，然后把结果显示回来。

比如你输入：

```bash
python --version
```

电脑会告诉你 Python 的版本。

## 跟着做：检查 Python

先打开终端，进入项目目录。

如果你已经在项目目录里，可以直接运行：

```bash
python3 --version
```

你也可以试试：

```bash
python --version
```

## 你应该看到什么结果

如果正常，你会看到类似：

```text
Python 3.11.8
```

版本号不一定和这里一样。

建议使用 Python 3.10 或更新版本。

如果提示 `command not found`，说明电脑找不到对应命令。你需要先安装 Python，或者确认 Python 命令是不是叫 `python3`。

## 跟着做：创建虚拟环境

在项目根目录运行：

```bash
python3 -m venv .venv
```

这条命令会创建一个叫 `.venv` 的文件夹。

`.venv` 就是当前项目的虚拟环境。

## 跟着做：激活虚拟环境

macOS 或 Linux 运行：

```bash
source .venv/bin/activate
```

Windows PowerShell 运行：

```powershell
.\.venv\Scripts\Activate.ps1
```

激活成功后，终端前面通常会出现：

```text
(.venv)
```

这表示你已经进入当前项目的专属 Python 环境。

## 常见错误

### 错误 1：`python3: command not found`

说明电脑找不到 `python3`。

可以试试：

```bash
python --version
```

如果还是不行，需要先安装 Python。

### 错误 2：激活虚拟环境后没有任何变化

有些终端主题不会显示 `(.venv)`。

你可以运行：

```bash
which python
```

如果路径里包含 `.venv`，说明已经激活。

### 错误 3：不小心把 `.venv` 提交到 GitHub

不要提交 `.venv`。

这个项目已经在 `.gitignore` 里忽略了 `.venv/`，正常情况下 Git 不会跟踪它。

## 小结

这一节你只需要记住：

- **Python**：写示例代码的语言
- **终端**：输入命令的地方
- **虚拟环境**：当前项目自己的 Python 小房间
