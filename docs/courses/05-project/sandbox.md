# Sandbox：让 Agent 在安全环境里试错

## 这一节你会学到什么

你会理解 Sandbox 是什么。

你还会知道什么时候应该让 Agent 在隔离环境里运行，而不是直接在真实电脑环境里操作。

## 一句话讲清楚

Sandbox 是给 Agent 练习和试错的安全空间。

用“电脑里的人”来理解：

> 如果助理要实验一段代码、安装依赖、运行命令，不要让他直接在你的主电脑上乱试，而是给他一个临时练习房间。

## 为什么需要它

到目前为止，我们主要让 Agent 做这些事：

- 生成学习计划。
- 保存笔记。
- 整理复盘问题。

这些风险相对低。

但如果你以后让 Agent 做这些事，风险就高多了：

- 执行 shell 命令。
- 安装依赖包。
- 运行不熟悉的代码。
- 修改项目文件。
- 调用系统工具。

如果没有隔离环境，出错可能影响你的真实项目或电脑。

Sandbox 的作用是：

- 让 Agent 在受控环境里试错。
- 即使出错，也尽量不影响真实环境。

## 用一个简单例子理解

你要教一个人做实验。

你不会让他直接在厨房随便混合化学品。

你会给他：

- 实验台。
- 防护规则。
- 可用材料。
- 失败后可以清理的环境。

Agent 的 sandbox 就是这种实验台。

## 回到 Agent

Deep Agents 可以和 sandbox 能力结合。

刚入门时，你只需要知道：

| 场景 | 是否需要 sandbox |
| --- | --- |
| 只回答问题 | 通常不需要 |
| 保存学习笔记 | 可以先用受限工作区 |
| 运行用户提供的代码 | 建议需要 |
| 安装依赖、执行命令 | 建议需要 |
| 操作生产数据库 | 必须非常谨慎，通常还要人工确认 |

本课程主线不强制你配置远程 sandbox。

因为不同 sandbox 服务可能需要额外账号、Key 或部署环境。

这一节的目标是先建立判断标准：

> 一旦 Agent 能执行代码或命令，就要考虑 sandbox。

## 代码里长什么样

不同 sandbox 后端的配置方式可能不同。

你以后会在官方文档里看到类似这种结构：

```python
from deepagents import create_deep_agent

agent = create_deep_agent(
    model="openai:gpt-4o-mini",
    system_prompt="你是一个安全谨慎的代码实验助手。",
    # sandbox=some_sandbox_backend,
)
```

这里的重点不是背 API，而是理解位置：

> Agent 如果要运行代码或命令，需要一个受控执行环境。

在本课程里，我们暂时不用远程 sandbox。

我们先用两个低风险替代做法：

1. 只让 Agent 在 `agent4all_workspace/` 里读写学习文件。
2. 不让 Agent 执行任意 shell 命令。

这不是生产级隔离，但足够适合入门练习。

## 跟着做

现在做一个判断练习。

下面哪些任务应该考虑 sandbox？

1. 解释什么是 memory。
2. 把学习计划保存到 `agent4all_workspace/`。
3. 运行用户给的一段 Python 脚本。
4. 安装一个未知依赖包并运行测试。
5. 删除 `agent4all_workspace/` 里的旧笔记。

参考答案：

- 1 通常不需要。
- 2 用受限工作区就可以。
- 3 建议需要 sandbox。
- 4 建议需要 sandbox。
- 5 至少需要 human-in-the-loop。

## 你应该看到什么结果

学完这一节，你应该能判断：

- **Agent 只是读写学习文件**：权限边界 + 人工确认通常够用。
- **Agent 要运行代码或系统命令**：应该考虑 sandbox。
- **Agent 要影响真实用户、真实钱、真实生产数据**：必须更谨慎。

## 常见错误

### 错误 1：以为本地项目目录就是 sandbox

不是。

`agent4all_workspace/` 只是一个受限工作区。

真正的 sandbox 通常会隔离文件系统、网络、进程或运行权限。

### 错误 2：觉得自己只是学习，不需要安全意识

学习阶段更适合养成安全习惯。

一开始就给 Agent 明确边界，后面做真实项目时会少踩很多坑。

### 错误 3：让 Agent 执行看不懂的命令

如果你看不懂一条命令，不要直接让 Agent 执行。

先让它解释：

- 这条命令会做什么？
- 会修改哪些文件？
- 失败后怎么恢复？

### 错误 4：把 sandbox 当成万能保险

Sandbox 能降低风险，但不能代替：

- 权限控制。
- 人工确认。
- 日志记录。
- 结果检查。
