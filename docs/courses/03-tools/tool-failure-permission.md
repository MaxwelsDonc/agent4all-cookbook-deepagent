# 工具失败与权限边界

## 这一节你会学到什么

你会理解一个很重要的问题：

> Agent 能使用工具，不代表它应该被允许做任何事。

这一节会讲：

- 工具为什么会失败。
- 工具失败时应该返回什么。
- 文件工具为什么要限制目录。
- 资料推荐工具为什么要说明来源和边界。
- 第三章应该守住哪些权限边界。

## 一句话讲清楚

工具越能做真实事情，越需要清楚的失败提示和权限边界。

## 为什么工具会失败

工具失败很正常。

常见原因包括：

- 用户给的参数不对。
- 文件名不安全。
- 目标文件夹不存在。
- 搜索不到资料。
- 模型传参和你预期的不一样。
- 外部 API 暂时不可用。

新手很容易以为：

> 工具失败 = Agent 坏了。

不是。

工具失败是 Agent 实践里很常见的一部分。

关键是让失败变得可理解、可修正。

这一章先用函数里的参数检查和返回提示处理常见失败。

更完整的错误处理可以在后面再接入中间检查层、人工确认和更严格的权限控制。

这里的中间检查层，后面也可能会看到英文名 middleware。

现在你只要先理解成：

> 工具真正执行前后，额外做一层检查。

## 先看安全文件工具

上一节的 `save_study_plan()` 里有几行代码很重要：

```python
safe_name = filename.strip().replace(" ", "_")

if not safe_name:
    return "文件名不能为空。"

if "/" in safe_name or "\\" in safe_name or safe_name.startswith("."):
    return "文件名不安全，请只使用普通文件名。"
```

这些代码不是为了好看。

它们是在设置边界：

- 不允许空文件名。
- 不允许路径分隔符。
- 不允许隐藏文件名。
- 只写入 `learning_notes` 目录。

也就是说，MaxLance Assistant 可以保存学习计划，但不能随便写你的电脑。

## 权限边界是什么

Permission 可以先理解成：

> 你允许 Agent 做什么，不允许它做什么。

在第三章，我们先用最简单的方式理解权限：

| 工具 | 可以做什么 | 不应该做什么 |
| --- | --- | --- |
| 计算工具 | 根据参数计算学习时间 | 修改文件、访问网络 |
| 推荐资料工具 | 从本地资料库返回建议 | 假装查过真实互联网 |
| 保存计划工具 | 写入 `learning_notes` 目录 | 写入任意路径、删除文件 |

工具边界越清楚，Agent 越安全。

## 让失败提示更清楚

坏的失败提示是：

```text
error
```

用户不知道发生了什么。

Agent 也很难继续帮助用户。

更好的失败提示是：

```text
文件名不安全，请只使用普通文件名。
```

这样 Agent 可以继续说：

> 你给的文件名包含路径符号。请换成 chapter3_plan.md 这样的普通文件名。

## 跟着做：故意触发一次失败

把用户消息临时改成：

```python
"content": (
    "请把我的学习计划保存为 ../secret.md。"
    "内容是：第三章学习工具调用。"
)
```

再运行：

```bash
python maxlance_agent.py
```

## 你应该看到什么结果

你应该看到 Agent 没有成功保存到 `../secret.md`。

工具应该返回类似信息：

```text
文件名不安全，请只使用普通文件名。
```

这说明边界生效了。

这不是坏事。

这是安全设计在工作。

测试完以后，把文件名改回普通名字：

```text
chapter3_plan.md
```

## 资料推荐工具也需要边界

上一节的 `recommend_learning_resource()` 只是本地资料推荐。

它不是真实互联网搜索。

所以它找不到内容时，应该诚实返回：

```text
暂时没有找到精确资料，建议先从课程总览开始复习。
```

不要让工具假装自己查过所有资料。

以后如果你接入真实搜索工具，也要考虑：

- 搜索来源是否可靠。
- 是否需要 API Key。
- 是否会产生费用。
- 是否会把用户输入发送到外部服务。
- 搜索失败时怎么告诉用户。

这些问题会在后面的安全和完整项目里继续展开。

## 第三章的边界

这一章只做到：

- 让 Agent 调用 Python 函数。
- 让 Agent 使用计算、推荐资料、保存计划这三类工具。
- 理解工具失败。
- 给有副作用的工具设置简单边界。

这一章不做：

- 不让 Agent 随便读写整个文件系统。
- 不接真实互联网搜索 API。
- 不做多 Agent 协作。
- 不做复杂权限系统。
- 不让工具删除文件或执行危险命令。

这不是能力不足。

这是学习顺序。

先把工具调用学明白，再进入更复杂的 DeepAgent 能力。

## 常见错误

### 错误 1：把所有能力都做成工具

不要一开始就给 Agent 十几个工具。

工具越多，越难判断问题出在哪里。

第三章先保持少量工具。

### 错误 2：文件工具没有限制路径

不要让新手示例写成：

```python
path = Path(filename)
path.write_text(content)
```

这会让工具可以写到不该写的位置。

先固定写入 `learning_notes`。

### 错误 3：工具失败后只报 Python 错误

原始 Python 错误对小白不友好。

尽量返回用户能理解的提示。

比如：

```text
每天学习时间必须大于 0。
```

比一长串 traceback 更适合教学。

### 错误 4：把权限问题留到最后

权限不是上线前才考虑。

从第一个有副作用的工具开始，就应该想：

> 这个工具允许 Agent 改变什么？

## 本章自测

试着回答下面几个问题：

1. Tool 和 Tool Calling 有什么区别？
2. 为什么工具函数最好有清楚的 docstring？
3. 工具返回值会去哪里？
4. 为什么保存文件工具要限制目录？
5. 如果 Agent 没有调用工具，你会先检查哪几件事？

如果你能用自己的话回答这些问题，第三章就算真正学到了。

## 小结

这一章你让 MaxLance Assistant 从“会说话”变成了“会做事”。

它现在可以：

- 调用计算工具。
- 推荐学习资料。
- 保存学习计划。
- 在工具失败时给出更清楚的提示。
- 对有副作用的工具保持基本边界。

下一章，我们会继续增强 MaxLance Assistant。

它不只是调用单个工具，而是开始处理更复杂的任务：规划、读写文件、管理上下文，并为后面的多 Agent 协作打基础。
