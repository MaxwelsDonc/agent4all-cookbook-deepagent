# 为什么 Agent 需要工具

## 这一节你会学到什么

你会先理解一件事：

> 模型很会思考和表达，但它不能凭空完成所有事情。

它需要工具。

这一节会解释：

- Tool 是什么。
- Tool Calling 是什么。
- 为什么工具会让 Agent 从“会说话”变成“会做事”。
- MaxLance Assistant 为什么需要工具。

## 一句话讲清楚

Tool 是 Agent 可以使用的能力。

Tool Calling 是 Agent 决定使用某个工具，并把参数交给工具执行。

## 为什么需要它

只靠模型本身，Agent 很容易遇到几个限制：

- 它不能真正读取你电脑上的文件。
- 它不能真正保存学习计划。
- 它不适合可靠地完成精确计算。
- 它不能自己访问搜索、数据库或其他外部系统。
- 它可能会把不知道的内容说得像真的一样。

一个再聪明的人，也不能凭空知道今天的天气。

他需要打开天气 App。

对 Agent 来说，天气 App、计算器、文件保存函数、搜索函数，都可以是工具。

## 用一个简单例子理解

假设你请一个学习助理帮你安排 DeepAgent 学习。

如果他只靠脑子，他可以说：

> 你可以每天学一点。

这听起来没错，但不够具体。

如果他有一个可靠的计算工具，他就可以先算：

> 5 天，每天 1.5 小时，一共是 7.5 小时。

然后再回答：

> 你有 7.5 小时，可以把前 2 小时用来复习 Agent 基础，后 5.5 小时用来跑代码和改工具。

这就是工具带来的变化。

Agent 不只是生成一段话，而是先做了一步可验证的操作。

## 回到 Agent

在 Agent 里，可以这样理解：

| 概念 | 简单理解 |
| --- | --- |
| Tool | Agent 可以使用的工具，比如计算器、搜索、保存文件 |
| Tool Calling | Agent 决定调用工具的动作 |
| 参数 | Agent 交给工具的信息 |
| 返回值 | 工具做完后交还给 Agent 的结果 |
| 上下文 | Agent 当前能看到并继续参考的信息 |

工具返回值会回到 Agent 的上下文里。

也就是说，Agent 可以先使用工具，再基于工具结果继续回答你。

## 工具调用流程

一个典型流程是：

现在不用背这个流程。

你只要记住：

> Agent 不是每次都调用工具。它会根据任务判断是否需要工具。

## MaxLance Assistant 为什么需要工具

MaxLance Assistant 是一个 DeepAgent 学习助理。

如果它只有聊天能力，它可以解释概念。

如果它有工具，它就可以做更多事：

- 帮你计算学习总时长。
- 根据主题推荐学习资料。
- 把学习计划保存成文件。
- 后面还可以把任务拆成步骤，保存笔记，生成练习题。

第三章先从最小工具开始。

不要急着做复杂系统。

我们先让它学会：

> 看到任务，选择工具，拿到结果，再回答用户。

## 常见误解

### 误解 1：工具越多越好

不是。

工具越多，Agent 越需要判断该用哪个，也越容易误用。

新手阶段先用少量、清晰、容易验证的工具。

### 误解 2：工具就是让 Agent 想干什么就干什么

不是。

工具应该有边界。

比如文件工具只允许写入固定目录，不应该随便删除电脑里的文件。

### 误解 3：Agent 一定会自动调用工具

不一定。

模型会根据任务、工具说明、系统提示词来判断是否调用工具。

如果工具描述不清楚，Agent 可能不用工具，或者用错工具。

## 小结

这一节你只需要记住：

- Tool 是 Agent 可以使用的能力。
- Tool Calling 是 Agent 决定使用工具的过程。
- 工具返回值会进入 Agent 上下文。
- MaxLance Assistant 会从一个会聊天的学习助理，慢慢变成会做事的学习助理。
