# 什么是 Agent

## 这一节你会学到什么

你会先建立一个最重要的直觉：

> Agent 可以理解成一个住在电脑里的“人”。

这个说法不是严格定义，但对小白很有用。

## 一句话讲清楚

Agent 是一个由大模型驱动、可以根据任务思考、使用工具并执行步骤的软件助手。

换句话说，聊天机器人主要是在“回答你”，Agent 更强调“帮你完成一件事”。

## 为什么需要 Agent

大模型本身很擅长理解语言和生成文字，但它也有明显限制：

- 它不知道你电脑里的文件，除非你把文件给它。
- 它不能自己打开网页、运行命令、写入文件，除非你给它工具。
- 它容易在长任务里忘记前面做过什么，除非系统帮它管理上下文。
- 它不会天然知道哪些操作危险，除非你设置权限和人工确认。

Agent 的意义就是：在模型外面加上一套执行系统，让模型不只是聊天，而是能围绕目标一步一步行动。

## 用一个简单例子理解

普通聊天机器人更像一个只会说话的人。

你问它：

> 帮我制定一个 7 天学习 DeepAgent 的计划。

它可以直接回答你一段文字。

但 Agent 不只是说话。它更像一个在电脑里帮你做事的人。它可以：

- 先问你有没有 Python 基础。
- 根据你的时间安排拆解学习路径。
- 把计划写进文件。
- 搜索资料。
- 根据你的反馈修改计划。

这就是我们这一章的贯穿例子：

> 一个帮助小白学习 DeepAgent 的学习助手。

如果它只会回答“什么是 DeepAgent”，它更像聊天机器人。

如果它能问你的基础、安排学习路径、保存计划、根据反馈调整，它就更像 Agent。

## 回到 Agent

所以学习 Agent 时，我们关心的不是“它能不能聊天”，而是它怎么完成任务。

一个最简单的 Agent 循环大概是这样：

<div class="a4a-flow">
  <div><b>用户提出任务</b></div>
  <div><b>Agent 理解任务</b></div>
  <div><b>制定下一步计划</b></div>
  <div><b>需要工具吗？</b><span>不需要 → 直接返回结果；需要 → 继续下一步</span></div>
  <div><b>调用工具，观察结果</b><span>↻ 然后回到「制定下一步计划」，直到任务可以收尾</span></div>
  <div class="is-end"><b>返回结果</b></div>
</div>

这里的关键是循环。

普通聊天通常是「你问一次，模型答一次」。

Agent 更像是一个循环：

1. 理解任务
2. 决定下一步
3. 必要时使用工具
4. 观察结果
5. 继续决定下一步，直到任务完成

## Agent 由哪些部分组成

先记住这几个最常见的部件就够了：

| 部件 | 用“电脑里的人”理解 | 在 Agent 里是什么意思 |
| --- | --- | --- |
| Model | 脑子 | 负责理解、推理和生成内容 |
| Prompt | 你说的话 | 你给 Agent 的任务或要求 |
| System Prompt | 岗位说明 | 规定 Agent 的身份、风格和工作规则 |
| Messages | 聊天记录 | 你和 Agent 到目前为止的对话 |
| Context | 当前能看到的信息 | Agent 做判断时能参考的资料 |
| Tools | 电脑工具 | 搜索、计算、读写文件、调用 API 等能力 |
| Permission | 允许范围 | 哪些事能做，哪些事要先问人 |

## 代码里长什么样

后面你会看到类似这样的代码：

```python
from deepagents import create_deep_agent

agent = create_deep_agent(
    model="...",
    tools=[],
    system_prompt="你是一个耐心的新手教练。"
)

result = agent.invoke({
    "messages": [
        {"role": "user", "content": "帮我制定一个 7 天学习 DeepAgent 的计划"}
    ]
})
```

现在不需要完全看懂。你只需要先知道：

- `model` 是这个“电脑里的人”的脑子。
- `tools` 是他能用的工具。
- `system_prompt` 是你给他的身份和工作规则。
- `messages` 是你和他的聊天记录。

这段代码里，真正让它成为 Agent 的，不只是 `model`。更重要的是：它可以围绕 `messages` 里的任务，结合 `tools` 和系统规则去执行。

## 跟着做

现在不用写代码，只做一个判断练习。

下面三句话里，哪些更像普通聊天，哪些更像 Agent 任务？

1. 解释一下什么是 DeepAgent。
2. 帮我查资料、整理成表格，并保存到一个文件里。
3. 根据我的时间安排，制定学习计划，之后每天根据反馈调整。

参考判断：

- 第 1 个更像普通聊天：它主要需要回答。
- 第 2 个更像 Agent 任务：它需要查资料、整理、保存文件。
- 第 3 个更像 Agent 任务：它需要长期跟进、根据反馈调整。

这个练习想让你看到：Agent 的重点不是“能说”，而是“能围绕目标做事”。

## 你应该看到什么结果

学完这一节，你应该能用自己的话说出：

> Agent 不是一个神秘的新物种，而是一个由大模型驱动、能使用工具和执行步骤的软件助手。

你也应该能分辨：

- 只需要回答的问题，普通聊天模型就够了。
- 需要多步骤、工具、文件、反馈和权限的任务，更适合交给 Agent。

## 常见误解

### Agent 不是魔法

Agent 不是真的人，也不会凭空知道一切。

它能做什么，取决于：

- 模型能力。
- 你给它的指令。
- 你提供的工具。
- 它能看到的上下文。
- 你允许它执行的操作。

### Agent 不等于大模型

大模型更像脑子。

Agent 是把脑子、工具、记忆、规则和执行流程组合起来的软件系统。

### Agent 不一定每次都需要很多工具

有些任务只需要回答问题，一个普通聊天模型就够了。

当任务需要查资料、读文件、写文件、分步骤执行、保存中间结果或等待人工确认时，Agent 的价值才更明显。
